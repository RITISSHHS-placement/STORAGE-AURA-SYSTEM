03 — Distributed Security Layer

AES-256-GCM encryption + Shamir's Secret Sharing distributes data across IPFS nodes. Steal 2 of 5 — get zero information.

AES-256-GCM

Military-Grade Encryption

C = AES_Encrypt(P, K) where: C = Ciphertext (encrypted output) P = Plaintext (original data) K = 256-bit key (PBKDF2-derived) IV = 96-bit random nonce Tag = 128-bit auth tag (GCM)
Key Size: 256 bits · Mode: GCM · Auth: AEAD
Shamir's Secret Sharing

(K, N) Threshold Scheme

P(x) = s + a₁x + a₂x² + ··· + aₖ₋₁xᵏ⁻¹ where: s = Secret (embedded at P(0)) aᵢ = Random coefficients Shares: {(i, P(i))} for i=1..N Prime p = 2¹²⁷ − 1 (Mersenne)
K shares → full recovery · K-1 shares → zero information
Reconstruction — Lagrange Interpolation
P(0) = Σᵢ yᵢ · Lᵢ(0) ← evaluates polynomial at secret point x=0 Lᵢ(0) = Πⱼ≠ᵢ (−xⱼ) / (xᵢ − xⱼ) (mod p) Security theorem: ∀ subsets S of size < K: H(secret | {shares in S}) = H(secret) (information-theoretically secure)
1
Encrypt with AES-256
Original data is encrypted with a 256-bit key derived from password via PBKDF2. Authentication tag prevents tampering.
2
Embed in Polynomial
Encrypted data becomes the constant term of a random K-degree polynomial over a Mersenne prime field.
3
Generate N Shares
Polynomial is evaluated at N points. Each evaluation is a share — distributed to separate IPFS nodes.
4
Reconstruct with K Shares
Lagrange interpolation reconstructs the polynomial from any K shares. The secret emerges at x=0.
